# 数据库与消息中间件设计

## 1. 数据库选型

| 类型 | 组件 | 用途 |
|---|---|---|
| 业务库 | PostgreSQL | 用户、车辆、货物、绑定、告警、指令 |
| 时序库 | TimescaleDB | GPS 点位、心跳历史 |
| 缓存 | Redis | 车辆最新位置、设备最新心跳、Token 黑名单 |
| 文件 | MinIO | 知识库文档、运单附件 |
| 向量 | pgvector | RAG 文档 chunk 向量 |

---

## 2. PostgreSQL 业务表

### 2.1 users

```sql
CREATE TABLE users (
  id VARCHAR(32) PRIMARY KEY,
  username VARCHAR(64) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  name VARCHAR(64) NOT NULL,
  role VARCHAR(32) NOT NULL,
  phone VARCHAR(32),
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

### 2.2 vehicles

```sql
CREATE TABLE vehicles (
  id BIGSERIAL PRIMARY KEY,
  plate VARCHAR(32) UNIQUE NOT NULL,
  vin_topic VARCHAR(64) UNIQUE NOT NULL,
  vehicle_type VARCHAR(64),
  capacity INT,
  driver_name VARCHAR(64),
  driver_phone VARCHAR(32),
  device_imei VARCHAR(32) UNIQUE NOT NULL,
  status VARCHAR(32) DEFAULT 'OFFLINE',
  device_status VARCHAR(32) DEFAULT 'OFFLINE',
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

### 2.3 cargo

```sql
CREATE TABLE cargo (
  cargo_id VARCHAR(64) PRIMARY KEY,
  cargo_type VARCHAR(64),
  weight NUMERIC(10,2),
  status VARCHAR(32) DEFAULT 'CREATED',
  origin_name VARCHAR(128),
  origin_lat DOUBLE PRECISION,
  origin_lng DOUBLE PRECISION,
  destination_name VARCHAR(128),
  destination_lat DOUBLE PRECISION,
  destination_lng DOUBLE PRECISION,
  loaded_at TIMESTAMPTZ,
  delivered_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

### 2.4 cargo_vehicle_binding

```sql
CREATE TABLE cargo_vehicle_binding (
  id VARCHAR(64) PRIMARY KEY,
  cargo_id VARCHAR(64) NOT NULL REFERENCES cargo(cargo_id),
  plate VARCHAR(32) NOT NULL REFERENCES vehicles(plate),
  status VARCHAR(32) DEFAULT 'ACTIVE',
  bound_at TIMESTAMPTZ DEFAULT now(),
  unbound_at TIMESTAMPTZ
);
CREATE UNIQUE INDEX ux_active_vehicle_binding ON cargo_vehicle_binding(plate) WHERE status = 'ACTIVE';
CREATE UNIQUE INDEX ux_active_cargo_binding ON cargo_vehicle_binding(cargo_id) WHERE status = 'ACTIVE';
```

### 2.5 cargo_status_logs

```sql
CREATE TABLE cargo_status_logs (
  id VARCHAR(64) PRIMARY KEY,
  cargo_id VARCHAR(64) NOT NULL,
  status VARCHAR(32) NOT NULL,
  lat DOUBLE PRECISION,
  lng DOUBLE PRECISION,
  remark TEXT,
  operator_id VARCHAR(32),
  created_at TIMESTAMPTZ DEFAULT now()
);
```

### 2.6 alerts

```sql
CREATE TABLE alerts (
  alert_id VARCHAR(64) PRIMARY KEY,
  alert_type VARCHAR(32) NOT NULL,
  severity VARCHAR(32) NOT NULL,
  status VARCHAR(32) NOT NULL DEFAULT 'PENDING',
  vehicle_plate VARCHAR(32),
  cargo_id VARCHAR(64),
  title VARCHAR(128),
  summary TEXT,
  description TEXT,
  lat DOUBLE PRECISION,
  lng DOUBLE PRECISION,
  triggered_at TIMESTAMPTZ NOT NULL,
  acknowledged_at TIMESTAMPTZ,
  resolved_at TIMESTAMPTZ,
  resolution VARCHAR(64),
  remark TEXT
);
```

### 2.7 alert_logs

```sql
CREATE TABLE alert_logs (
  id VARCHAR(64) PRIMARY KEY,
  alert_id VARCHAR(64) NOT NULL REFERENCES alerts(alert_id),
  operator_id VARCHAR(32),
  operator_name VARCHAR(64),
  action VARCHAR(64) NOT NULL,
  remark TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

### 2.8 commands

```sql
CREATE TABLE commands (
  command_id VARCHAR(64) PRIMARY KEY,
  plate VARCHAR(32) NOT NULL,
  vin_topic VARCHAR(64) NOT NULL,
  command_type VARCHAR(32) NOT NULL,
  priority VARCHAR(32) NOT NULL,
  payload JSONB,
  status VARCHAR(32) DEFAULT 'SENT',
  mqtt_topic VARCHAR(128),
  created_by VARCHAR(32),
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

### 2.9 command_logs

```sql
CREATE TABLE command_logs (
  id VARCHAR(64) PRIMARY KEY,
  command_id VARCHAR(64) NOT NULL REFERENCES commands(command_id),
  status VARCHAR(32) NOT NULL,
  source VARCHAR(32),
  raw_payload JSONB,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

### 2.10 knowledge_documents / knowledge_chunks

```sql
CREATE TABLE knowledge_documents (
  document_id VARCHAR(64) PRIMARY KEY,
  title VARCHAR(255) NOT NULL,
  category VARCHAR(64),
  object_key VARCHAR(512),
  status VARCHAR(32) DEFAULT 'UPLOADED',
  chunk_count INT DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE EXTENSION IF NOT EXISTS vector;
CREATE TABLE knowledge_chunks (
  chunk_id VARCHAR(64) PRIMARY KEY,
  document_id VARCHAR(64) REFERENCES knowledge_documents(document_id),
  chunk_index INT,
  content TEXT,
  embedding VECTOR(1024),
  created_at TIMESTAMPTZ DEFAULT now()
);
CREATE INDEX idx_knowledge_chunks_embedding ON knowledge_chunks USING ivfflat (embedding vector_cosine_ops);
```

## 3. TimescaleDB 时序表

### 3.1 gps_points

```sql
CREATE TABLE gps_points (
  time TIMESTAMPTZ NOT NULL,
  vehicle_id VARCHAR(32) NOT NULL,
  cargo_id VARCHAR(64),
  imei VARCHAR(32),
  lat DOUBLE PRECISION,
  lng DOUBLE PRECISION,
  speed SMALLINT,
  heading SMALLINT,
  accuracy DOUBLE PRECISION
);
SELECT create_hypertable('gps_points', 'time', if_not_exists => TRUE);
CREATE INDEX idx_gps_vehicle_time ON gps_points(vehicle_id, time DESC);
```

### 3.2 device_heartbeats

```sql
CREATE TABLE device_heartbeats (
  time TIMESTAMPTZ NOT NULL,
  imei VARCHAR(32) NOT NULL,
  plate VARCHAR(32),
  battery INT,
  signal INT,
  gnss_satellites INT,
  temp DOUBLE PRECISION
);
SELECT create_hypertable('device_heartbeats', 'time', if_not_exists => TRUE);
CREATE INDEX idx_heartbeat_imei_time ON device_heartbeats(imei, time DESC);
```

## 4. Redis Key 设计

| Key | 类型 | 说明 |
|---|---|---|
| `logistics:vehicle:latest:{plate}` | HASH/JSON | 车辆最新位置 |
| `logistics:device:heartbeat:{imei}` | HASH/JSON | 设备最新心跳 |
| `logistics:auth:blacklist:{token}` | STRING | 已登出 token 黑名单 |
| `logistics:ws:sub:{userId}` | SET | 用户订阅车辆 |
| `logistics:alert:dedup:{plate}:{type}` | STRING | 告警去重 |
