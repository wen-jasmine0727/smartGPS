# 智慧物流 IoT 平台 · 接口文档扩展完整版

> Base URL：`http://localhost:8080/api/v1`  
> Swagger/Knife4j：`http://localhost:8080/doc.html`，页面应只显示 `智慧物流接口`  
> WebSocket：`ws://localhost:8080/api/v1/ws?token=<JWT>`  
> MQTT Broker：`mqtt://localhost:1883`，如做 TLS 则使用 `mqtts://localhost:8883`  
> 数据格式：JSON / UTF-8  
> 时间格式：ISO 8601 UTC，例如 `2026-06-29T06:15:00Z`  
> 坐标：WGS-84，统一字段 `lat`、`lng`

---

## 0. 版本约束

| 项 | 约定 |
|---|---|
| API 前缀 | `/api/v1` |
| 认证方式 | `Authorization: Bearer <JWT_TOKEN>` |
| 分页页码 | `page` 从 1 开始 |
| 成功 code | `0` |
| 通用错误 | `40001/40101/40301/40401/40901/50001` |
| 车牌展示格式 | `沪A·C0291` |
| MQTT Topic 车牌格式 | `沪A-C0291`，将 `·` 替换为 `-` |

### 0.1 Swagger/Knife4j 调试规则

1. 打开 `http://localhost:8080/doc.html`。
2. 页面应只显示 `智慧物流接口`。
3. 先调用 `POST /api/v1/auth/login`，账号可用 `dispatcher / 123456`。
4. 复制返回的 `data.accessToken`。
5. 调需要登录的接口时，在 `Authorization` 请求头填写 `Bearer <accessToken>`。

如果接口文档页面打不开，先确认后端已启动、Maven 已重新加载，并且 `application.yml` 中启用了 `spring.mvc.pathmatch.matching-strategy=ant_path_matcher`。如果仍看到 C 端/员工/分类接口，说明后端没有重启或浏览器缓存了旧页面。

### 11.2 Topic 总表

|---|---|---|---|

### 11.3 GPSData

```json
{
  "imei": "861234567890123",
  "ts": 1719634500,
  "lat": 30.4219,
  "lng": 120.5738,
  "speed": 68,
  "heading": 225,
  "accuracy": 3.5
}
```

处理要求：

```txt
1. 校验 imei 是否存在。
2. 通过 vinTopic 找到 vehicle。
3. 找到当前绑定 cargo。
4. 发送到 Kafka topic: gps-points。
5. Consumer 写入 TimescaleDB gps_points。
6. Redis 更新 logistics:vehicle:latest:{plate}。
7. WebSocket 推送 vehicle.position。
```
