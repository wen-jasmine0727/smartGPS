# 前端 2 · 登录、告警中心、车辆货物管理、设备在线、智能问答

## 1. 你的定位

你负责管理闭环页面：登录进系统、看告警、处理告警、管理车辆货物、查看设备在线、使用智能问答。

负责页面：

```txt
登录页
03_alert_log_center.html → Vue 页面：告警与日志中心
04_fleet_cargo_management.html → Vue 页面：车辆与货物管理台
智能问答页/弹窗
设备在线状态模块
```

---

## 2. 技术栈

- Vue 3
- Vite
- TypeScript
- Naive UI
- Pinia
- Axios
- WebSocket 告警推送可选接入

---

## 3. 登录与权限

### 接口

| 功能 | 接口 |
|---|---|
| 登录 | `POST /auth/login` |
| 当前用户 | `GET /users/me` |
| 登出 | `POST /auth/logout` |

### 要做

- 登录表单。
- token 保存。
- Axios 请求头自动带 token。
- 路由守卫。
- 根据 role 控制菜单显示。

---

## 4. 告警中心

### 必须做

- 告警统计卡片。
- 告警列表。
- 筛选：级别、类型、状态、车辆。
- 告警详情。
- 处理日志。
- 确认告警。
- 关闭告警。
- WebSocket 新告警提醒。

### 接口

| 功能 | 接口/协议 |
|---|---|
| 告警列表 | `GET /alerts` |
| 告警详情 | `GET /alerts/{alertId}` |
| 告警统计 | `GET /alerts/stats` |
| 确认告警 | `POST /alerts/{alertId}/acknowledge` |
| 关闭告警 | `POST /alerts/{alertId}/resolve` |
| 新告警推送 | `WebSocket alert.triggered` |

---

## 5. 车辆货物管理

### 必须做

- 车辆列表。
- 新增车辆表单。
- 修改车辆。
- 删除/解绑车辆。
- 货物列表。
- 新建货物。
- 绑定货物与车辆。
- 解绑货物。
- 上报货物状态。
- 设备在线状态。

### 接口

| 功能 | 接口 |
|---|---|
| 车辆列表 | `GET /vehicles` |
| 新增车辆 | `POST /vehicles` |
| 修改车辆 | `PUT /vehicles/{id}` |
| 删除车辆 | `DELETE /vehicles/{id}` |
| 货物列表 | `GET /cargo` |
| 新建货物 | `POST /cargo` |
| 货物详情 | `GET /cargo/{cargoId}` |
| 绑定 | `POST /cargo/bind` |
| 解绑 | `POST /cargo/unbind` |
| 状态上报 | `PUT /cargo/{cargoId}/status` |
| 设备在线 | `GET /devices/status` |

## 6. 智能问答

### 必须做

- 推荐问题展示。
- 输入问题。
- 展示回答。
- 展示 sources。
- 支持 sessionId。
- loading 状态。

### 接口

| 功能 | 接口 |
|---|---|
| 推荐问题 | `GET /assistant/suggestions` |
| 发送问题 | `POST /assistant/chat` |
| 历史消息 | `GET /assistant/sessions/{sessionId}/messages` |
| 上传知识库 | `POST /knowledge/documents`，可选 |
| 文档索引 | `POST /knowledge/documents/{documentId}/index`，可选 |

## 8. 完成标准

- 登录后能进入对应页面。
- 告警数据来自真实后端。
- 确认/关闭告警后状态真实变化。
- 新增车辆/货物/绑定能写入数据库。
- 设备在线状态来自真实 heartbeat。
- 智能问答返回真实 answer + sources。

---

## 9. 避坑

1. 不要在前端写死告警数据。
2. 表单字段必须和接口文档一致。
3. 关闭告警需要 resolution 和 remark。
4. 智能问答要展示 sources，否则看不出 RAG。
5. token 过期要能回登录页。
