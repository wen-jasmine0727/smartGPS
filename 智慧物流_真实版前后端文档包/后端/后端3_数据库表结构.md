# 后端3 数据库表结构

## 1. 表概览

| 表名 | 说明 | 主要接口 |
|---|---|---|
| `alerts` | 告警记录（偏航/停车/开箱/离线） | `/alerts`、`/alerts/{alertId}`、`/alerts/stats` |
| `alert_logs` | 告警处理日志（确认/关闭） | `/alerts/{alertId}/acknowledge`、`/alerts/{alertId}/resolve` |
| `commands` | 调度指令（改道/停靠/加急/返程） | `/vehicles/{plate}/command`、`/vehicles/{plate}/commands` |
| `command_logs` | 指令状态流转日志 | `/vehicles/{plate}/command/{commandId}` |
| `knowledge_documents` | 知识库文档（路线/安全/维修/法规） | `/knowledge/documents`（上传/列表/删除） |
| `knowledge_chunks` | 文档切片 + pgvector 向量 | `/knowledge/documents/{documentId}/index`（切片与向量化） |

---

## 2. 表关系

```txt
vehicles
  ├─ alerts.vehicle_plate
  └─ commands.plate

cargo
  └─ alerts.cargo_id

alerts
  └─ alert_logs.alert_id

commands
  └─ command_logs.command_id

knowledge_documents
  └─ knowledge_chunks.document_id
```

核心规则：

| 规则 | 说明 |
|---|---|
| 告警去重 | 同一 `vehicle_plate` + `alert_type` 在 `status IN ('PENDING','ACKNOWLEDGED')` 时不重复生成 |
| 离线判定 | `device_heartbeats` 最近一条心跳距今 > 90 秒触发 `DEVICE_OFFLINE` |
| 指令状态 | 必须由后端收到 MQTT command/ack 后真实更新 `command_logs`，不能前端假改 |
| 车牌双字段 | `commands` 表同时保存 `plate`（展示）和 `vin_topic`（MQTT Topic），同 vehicles 规则 |
| 向量维度 | `knowledge_chunks.embedding` 固定为 `VECTOR(1024)`，使用 bge-m3 embedding 模型 |
| RAG 检索 | ivfflat 索引 + vector_cosine_ops，Top-5 召回后拼接 Prompt 调用 LLM |

## 3. alerts 告警表

### 3.1 建表 SQL

```sql
CREATE TABLE alerts (
  alert_id VARCHAR(64) PRIMARY KEY,
  alert_type VARCHAR(32) NOT NULL,
  severity VARCHAR(32) NOT NULL,
  status VARCHAR(32) NOT NULL DEFAULT 'PENDING',
  vehicle_plate VARCHAR(32),
  cargo_id VARCHAR(64),
  title VARCHAR(128),
  summary TEXT,
  description TEXT,
  lat DOUBLE PRECISION,
  lng DOUBLE PRECISION,
  triggered_at TIMESTAMPTZ NOT NULL,
  acknowledged_at TIMESTAMPTZ,
  resolved_at TIMESTAMPTZ,
  resolution VARCHAR(64),
  remark TEXT
);

CREATE INDEX idx_alerts_status ON alerts(status);
CREATE INDEX idx_alerts_vehicle ON alerts(vehicle_plate);
CREATE INDEX idx_alerts_triggered ON alerts(triggered_at DESC);
```

### 3.2 字段说明

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `alert_id` | `VARCHAR(64)` | PK | 告警 ID，例如 `ALT-20260629-001` |
| `alert_type` | `VARCHAR(32)` | NOT NULL | 告警类型 |
| `severity` | `VARCHAR(32)` | NOT NULL | 严重程度 |
| `status` | `VARCHAR(32)` | NOT NULL, DEFAULT `PENDING` | 处理状态 |
| `vehicle_plate` | `VARCHAR(32)` |  | 展示车牌，例如 `沪A·C0291` |
| `cargo_id` | `VARCHAR(64)` |  | 关联货物 ID |
| `title` | `VARCHAR(128)` |  | 告警标题 |
| `summary` | `TEXT` |  | 一句话摘要 |
| `description` | `TEXT` |  | 详细描述 |
| `lat` | `DOUBLE PRECISION` |  | 触发时纬度（WGS-84） |
| `lng` | `DOUBLE PRECISION` |  | 触发时经度（WGS-84） |
| `triggered_at` | `TIMESTAMPTZ` | NOT NULL | 触发时间（UTC ISO 8601） |
| `acknowledged_at` | `TIMESTAMPTZ` |  | 确认时间 |
| `resolved_at` | `TIMESTAMPTZ` |  | 关闭时间 |
| `resolution` | `VARCHAR(64)` |  | 处理方式 |
| `remark` | `TEXT` |  | 关闭备注 |

### 3.3 枚举

| 字段 | 可选值 | 说明 |
|---|---|---|
| `alert_type` | `ROUTE_DEVIATION` | 偏航：偏离预设路线 > 3km 且持续 > 10 分钟 |
| `alert_type` | `ABNORMAL_STOP` | 异常停车：速度为 0 持续 > 15 分钟且不在装卸点 |
| `alert_type` | `CARGO_DOOR` | 开箱：DOOR_OPEN 事件且不在装卸点 |
| `alert_type` | `DEVICE_OFFLINE` | 设备离线：lastHeartbeat > 90 秒 |
| `severity` | `CRITICAL` | 严重 |
| `severity` | `WARNING` | 警告 |
| `severity` | `INFO` | 提示 |
| `status` | `PENDING` | 待处理 |
| `status` | `ACKNOWLEDGED` | 已确认 |
| `status` | `RESOLVED` | 已关闭 |
| `resolution` | `REROUTED` | 已改道 |
| `resolution` | `RETURNED` | 已返程 |
| `resolution` | `ACKNOWLEDGED` | 已知悉 |
| `resolution` | `IGNORED` | 忽略（误报） |

## 5. commands 调度指令表

### 5.1 建表 SQL

```sql
CREATE TABLE commands (
  command_id VARCHAR(64) PRIMARY KEY,
  plate VARCHAR(32) NOT NULL,
  vin_topic VARCHAR(64) NOT NULL,
  command_type VARCHAR(32) NOT NULL,
  priority VARCHAR(32) NOT NULL,
  payload JSONB,
  status VARCHAR(32) DEFAULT 'SENT',
  mqtt_topic VARCHAR(128),
  created_by VARCHAR(32),
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_commands_plate ON commands(plate, created_at DESC);
```

### 5.2 字段说明

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `command_id` | `VARCHAR(64)` | PK | 指令 ID，例如 `CMD-20260629-001` |
| `plate` | `VARCHAR(32)` | NOT NULL | 展示车牌，例如 `沪A·C0291` |
| `vin_topic` | `VARCHAR(64)` | NOT NULL | MQTT Topic 车牌，例如 `沪A-C0291` |
| `command_type` | `VARCHAR(32)` | NOT NULL | 指令类型 |
| `priority` | `VARCHAR(32)` | NOT NULL | 优先级 |
| `payload` | `JSONB` |  | 指令负载（新路线坐标、停靠点等） |
| `status` | `VARCHAR(32)` | DEFAULT `SENT` | 指令状态 |
| `mqtt_topic` | `VARCHAR(128)` |  | 下发的 MQTT Topic，例如 `vehicle/沪A-C0291/command` |
| `created_by` | `VARCHAR(32)` |  | 下发人 ID |
| `created_at` | `TIMESTAMPTZ` | DEFAULT now() | 创建时间 |
| `updated_at` | `TIMESTAMPTZ` | DEFAULT now() | 更新时间 |

### 5.3 枚举

| 字段 | 可选值 | 说明 |
|---|---|---|
| `command_type` | `REROUTE` | 改道：下发新路线坐标 |
| `command_type` | `STOP` | 停靠：要求在指定地点停靠 |

| `command_type` | `RETURN` | 返程：要求返回起点 |

| `status` | `SENT` | 已下发（Cloud -> EMQX） |
| `status` | `RECEIVED` | 设备已收到 |

## 8. knowledge_chunks 文档切片 + 向量表

### 8.1 建表 SQL

```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE knowledge_chunks (
  chunk_id VARCHAR(64) PRIMARY KEY,
  document_id VARCHAR(64) REFERENCES knowledge_documents(document_id),
  chunk_index INT,
  content TEXT,
  embedding VECTOR(1024),
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_knowledge_chunks_embedding
  ON knowledge_chunks
  USING ivfflat (embedding vector_cosine_ops);

CREATE INDEX idx_knowledge_chunks_doc
  ON knowledge_chunks(document_id);
```

### 8.2 字段说明

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `chunk_id` | `VARCHAR(64)` | PK | 切片 ID |
| `document_id` | `VARCHAR(64)` | FK | 关联文档 ID |
| `chunk_index` | `INT` |  | 切片序号（从 0 开始） |
| `content` | `TEXT` |  | 切片文本内容（300-800 字） |
| `embedding` | `VECTOR(1024)` |  | bge-m3 向量（1024 维） |
| `created_at` | `TIMESTAMPTZ` | DEFAULT now() | 创建时间 |

### 8.3 索引说明

| 索引 | 类型 | 说明 |
|---|---|---|
| `idx_knowledge_chunks_embedding` | ivfflat + vector_cosine_ops | 向量余弦相似度检索，RAG 问答使用 |
| `idx_knowledge_chunks_doc` | B-tree | 按文档 ID 查询所有切片 |

核心流程：

```txt
上传文档 -> MinIO
  -> 解析文本（Markdown/PDF/Word）
  -> 切片（300-800 字/块，按段落边界切割）
  -> 调用 embedding 模型（bge-m3）生成 VECTOR(1024)
  -> 写入 knowledge_chunks
  -> 用户提问 -> embedding 向量化 -> pgvector Top-5 检索
  -> 拼接 Prompt -> 调用 LLM -> 返回 answer + sources
```

## 10. 常用查询

### 10.1 查看待处理告警

```sql
SELECT alert_id, alert_type, severity, status, vehicle_plate, title, triggered_at
FROM alerts
WHERE status IN ('PENDING', 'ACKNOWLEDGED')
ORDER BY
  CASE severity WHEN 'CRITICAL' THEN 0 WHEN 'WARNING' THEN 1 ELSE 2 END,
  triggered_at DESC;
```

### 10.2 查看某告警的处理日志

```sql
SELECT id, operator_name, action, remark, created_at
FROM alert_logs
WHERE alert_id = 'ALT-20260629-001'
ORDER BY created_at DESC;
```

### 10.3 告警统计（按类型+状态）

```sql
SELECT alert_type, severity, status, count(*) AS cnt
FROM alerts
GROUP BY alert_type, severity, status
ORDER BY alert_type, severity, status;
```

### 10.4 查看某车辆的调度指令

```sql
SELECT command_id, command_type, priority, status, created_by, created_at
FROM commands
WHERE plate = '沪A·C0291'
ORDER BY created_at DESC;
```

### 10.5 查看某指令的状态流转

```sql
SELECT id, status, source, created_at
FROM command_logs
WHERE command_id = 'CMD-20260629-001'
ORDER BY created_at;
```

### 10.6 查看知识库文档与切片数

```sql
SELECT document_id, title, category, status, chunk_count, created_at
FROM knowledge_documents
ORDER BY created_at DESC;
```

### 10.7 RAG 语义检索（用问题向量检索 Top-5 切片）

```sql
SELECT c.chunk_id, c.document_id, d.title, c.content,
       c.embedding <=> '<question_embedding>'::VECTOR AS distance
FROM knowledge_chunks c
JOIN knowledge_documents d ON d.document_id = c.document_id
WHERE d.status = 'INDEXED'
ORDER BY c.embedding <=> '<question_embedding>'::VECTOR
LIMIT 5;
```

### 10.8 检查告警去重（某车某类型是否有未关闭告警）

```sql
SELECT alert_id, alert_type, status, triggered_at
FROM alerts
WHERE vehicle_plate = '沪A·C0291'
  AND alert_type = 'ROUTE_DEVIATION'
  AND status IN ('PENDING', 'ACKNOWLEDGED')
ORDER BY triggered_at DESC;
```
