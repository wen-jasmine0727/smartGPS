# Postgre表结构

| 表名 | 说明 | 主要接口 |
|---|---|---|
| `users` | 登录用户、角色、手机号 | `/auth/login`、`/users/me` |
| `vehicles` | 车辆、司机、设备 IMEI、车辆状态 | `/vehicles`、`/vehicles/{id}` |
| `cargo` | 货物、起点终点、状态 | `/cargo`、`/cargo/{cargoId}` |
| `cargo_vehicle_binding` | 货物和车辆绑定关系 | `/cargo/bind`、`/cargo/unbind` |
| `cargo_status_logs` | 货物状态流转日志 | `/cargo/{cargoId}/status-logs` |

---

## 2. 表关系

```txt
users
  └─ cargo_status_logs.operator_id

vehicles
  └─ cargo_vehicle_binding.plate

cargo
  ├─ cargo_vehicle_binding.cargo_id
  └─ cargo_status_logs.cargo_id
```

核心规则：

| 规则 | 说明 |
|---|---|
| 一车一货 | 同一时间一辆车只能有一个 `ACTIVE` 绑定 |
| 一货一车 | 同一时间一个货物只能有一个 `ACTIVE` 绑定 |
| 展示车牌 | 页面展示用 `plate`，例如 `沪A·C0291` |
| MQTT 车牌 | Topic 使用 `vin_topic`，例如 `沪A-C0291` |
| 密码存储 | `users.password_hash` 保存 MD5，不保存明文 |

## 3. users 用户表

### 3.1 建表 SQL

```sql
CREATE TABLE users (
  id VARCHAR(32) PRIMARY KEY,
  username VARCHAR(64) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  name VARCHAR(64) NOT NULL,
  role VARCHAR(32) NOT NULL,
  phone VARCHAR(32),
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

### 3.2 字段说明

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | `VARCHAR(32)` | PK | 用户 ID，例如 `USR-002` |
| `username` | `VARCHAR(64)` | UNIQUE, NOT NULL | 登录用户名 |
| `password_hash` | `VARCHAR(255)` | NOT NULL | 密码 MD5，例如 `123456` 的 MD5 |
| `name` | `VARCHAR(64)` | NOT NULL | 用户姓名 |
| `role` | `VARCHAR(32)` | NOT NULL | 角色 |
| `phone` | `VARCHAR(32)` |  | 手机号 |
| `created_at` | `TIMESTAMPTZ` | DEFAULT now() | 创建时间 |
| `updated_at` | `TIMESTAMPTZ` | DEFAULT now() | 更新时间 |

### 3.3 角色枚举

| 角色 | 说明 |
|---|---|
| `SHIPPER` | 货主 |
| `DISPATCHER` | 调度员 |
| `WAREHOUSE` | 仓管 |
| `ADMIN` | 管理员 |
| `DRIVER` | 司机 |

## 4. vehicles 车辆表

### 4.1 建表 SQL

```sql
CREATE TABLE vehicles (
  id BIGSERIAL PRIMARY KEY,
  plate VARCHAR(32) UNIQUE NOT NULL,
  vin_topic VARCHAR(64) UNIQUE NOT NULL,
  vehicle_type VARCHAR(64),
  capacity INT,
  driver_name VARCHAR(64),
  driver_phone VARCHAR(32),
  device_imei VARCHAR(32) UNIQUE NOT NULL,
  status VARCHAR(32) DEFAULT 'OFFLINE',
  device_status VARCHAR(32) DEFAULT 'OFFLINE',
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

如果本地已经按旧版本建过 `vehicles`，不要整库重建，执行：

```sql
\i sky-take-out/sky-server/src/main/resources/sql/backend1_add_vehicle_id.sql
```

### 4.2 字段说明

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | `BIGSERIAL` | PK | 车辆主键 ID，前端编辑、删除、详情使用 |
| `plate` | `VARCHAR(32)` | UNIQUE, NOT NULL | 展示车牌，例如 `沪A·C0291` |
| `vin_topic` | `VARCHAR(64)` | UNIQUE, NOT NULL | MQTT Topic 车辆标识，例如 `沪A-C0291` |
| `vehicle_type` | `VARCHAR(64)` |  | 车辆类型 |
| `capacity` | `INT` |  | 载重，单位 kg |
| `driver_name` | `VARCHAR(64)` |  | 司机姓名 |
| `driver_phone` | `VARCHAR(32)` |  | 司机手机号 |
| `device_imei` | `VARCHAR(32)` | UNIQUE, NOT NULL | GPS/IoT 设备 IMEI |
| `status` | `VARCHAR(32)` | DEFAULT `OFFLINE` | 车辆业务状态 |
| `device_status` | `VARCHAR(32)` | DEFAULT `OFFLINE` | 设备在线状态 |
| `created_at` | `TIMESTAMPTZ` | DEFAULT now() | 创建时间 |
| `updated_at` | `TIMESTAMPTZ` | DEFAULT now() | 更新时间 |

### 4.3 状态枚举

| 字段 | 可选值 | 说明 |
|---|---|---|
| `status` | `MOVING` | 行驶中 |
| `status` | `STOPPED` | 停止中 |
| `status` | `OFFLINE` | 离线 |
| `device_status` | `ONLINE` | 设备在线 |
| `device_status` | `OFFLINE` | 设备离线 |

## 5. cargo 货物表

### 5.1 建表 SQL

```sql
CREATE TABLE cargo (
  cargo_id VARCHAR(64) PRIMARY KEY,
  cargo_type VARCHAR(64),
  weight NUMERIC(10,2),
  status VARCHAR(32) DEFAULT 'CREATED',
  origin_name VARCHAR(128),
  origin_lat DOUBLE PRECISION,
  origin_lng DOUBLE PRECISION,
  destination_name VARCHAR(128),
  destination_lat DOUBLE PRECISION,
  destination_lng DOUBLE PRECISION,
  loaded_at TIMESTAMPTZ,
  delivered_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

### 5.2 字段说明

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `cargo_id` | `VARCHAR(64)` | PK | 货物 ID，例如 `SH-HZ-20260629-0291` |
| `cargo_type` | `VARCHAR(64)` |  | 货物类型 |
| `weight` | `NUMERIC(10,2)` |  | 重量，单位 kg |
| `status` | `VARCHAR(32)` | DEFAULT `CREATED` | 货物状态 |
| `origin_name` | `VARCHAR(128)` |  | 起点名称 |
| `origin_lat` | `DOUBLE PRECISION` |  | 起点纬度 |
| `origin_lng` | `DOUBLE PRECISION` |  | 起点经度 |
| `destination_name` | `VARCHAR(128)` |  | 终点名称 |
| `destination_lat` | `DOUBLE PRECISION` |  | 终点纬度 |
| `destination_lng` | `DOUBLE PRECISION` |  | 终点经度 |
| `loaded_at` | `TIMESTAMPTZ` |  | 装货时间 |
| `delivered_at` | `TIMESTAMPTZ` |  | 送达时间 |
| `created_at` | `TIMESTAMPTZ` | DEFAULT now() | 创建时间 |
| `updated_at` | `TIMESTAMPTZ` | DEFAULT now() | 更新时间 |

### 5.3 状态枚举

| 状态 | 说明 |
|---|---|
| `CREATED` | 已创建 |
| `LOADED` | 已装货 |
| `IN_TRANSIT` | 运输中 |
| `DELIVERED` | 已送达 |
| `CANCELLED` | 已取消 |

## 6. cargo_vehicle_binding 货物车辆绑定表

### 6.1 建表 SQL

```sql
CREATE TABLE cargo_vehicle_binding (
  id VARCHAR(64) PRIMARY KEY,
  cargo_id VARCHAR(64) NOT NULL REFERENCES cargo(cargo_id),
  plate VARCHAR(32) NOT NULL REFERENCES vehicles(plate),
  status VARCHAR(32) DEFAULT 'ACTIVE',
  bound_at TIMESTAMPTZ DEFAULT now(),
  unbound_at TIMESTAMPTZ
);

CREATE UNIQUE INDEX ux_active_vehicle_binding
  ON cargo_vehicle_binding(plate)
  WHERE status = 'ACTIVE';

CREATE UNIQUE INDEX ux_active_cargo_binding
  ON cargo_vehicle_binding(cargo_id)
  WHERE status = 'ACTIVE';
```

### 6.2 字段说明

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | `VARCHAR(64)` | PK | 绑定记录 ID |
| `cargo_id` | `VARCHAR(64)` | FK, NOT NULL | 货物 ID |
| `plate` | `VARCHAR(32)` | FK, NOT NULL | 车辆车牌 |
| `status` | `VARCHAR(32)` | DEFAULT `ACTIVE` | 绑定状态 |
| `bound_at` | `TIMESTAMPTZ` | DEFAULT now() | 绑定时间 |
| `unbound_at` | `TIMESTAMPTZ` |  | 解绑时间 |

### 6.3 状态枚举

| 状态 | 说明 |
|---|---|
| `ACTIVE` | 当前有效绑定 |
| `UNBOUND` | 已解绑 |

## 7. cargo_status_logs 货物状态日志表

### 7.1 建表 SQL

```sql
CREATE TABLE cargo_status_logs (
  id VARCHAR(64) PRIMARY KEY,
  cargo_id VARCHAR(64) NOT NULL REFERENCES cargo(cargo_id),
  status VARCHAR(32) NOT NULL,
  lat DOUBLE PRECISION,
  lng DOUBLE PRECISION,
  remark TEXT,
  operator_id VARCHAR(32),
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_cargo_status_logs_cargo_time
  ON cargo_status_logs(cargo_id, created_at DESC);
```

### 7.2 字段说明

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | `VARCHAR(64)` | PK | 日志 ID |
| `cargo_id` | `VARCHAR(64)` | FK, NOT NULL | 货物 ID |
| `status` | `VARCHAR(32)` | NOT NULL | 状态 |
| `lat` | `DOUBLE PRECISION` |  | 上报纬度 |
| `lng` | `DOUBLE PRECISION` |  | 上报经度 |
| `remark` | `TEXT` |  | 备注 |
| `operator_id` | `VARCHAR(32)` |  | 操作人 ID |
| `created_at` | `TIMESTAMPTZ` | DEFAULT now() | 创建时间 |

## 9. 常用查询

### 9.1 查看用户

```sql
SELECT id, username, name, role, phone
FROM users
ORDER BY id;
```

### 9.2 查看车辆列表

```sql
SELECT plate, vin_topic, vehicle_type, driver_name, device_imei, status, device_status
FROM vehicles
ORDER BY updated_at DESC;
```

### 9.3 查看当前有效绑定

```sql
SELECT b.id, b.cargo_id, c.cargo_type, b.plate, v.driver_name, b.status, b.bound_at
FROM cargo_vehicle_binding b
JOIN cargo c ON c.cargo_id = b.cargo_id
JOIN vehicles v ON v.plate = b.plate
WHERE b.status = 'ACTIVE';
```

### 9.4 查看货物状态日志

```sql
SELECT cargo_id, status, lat, lng, remark, operator_id, created_at
FROM cargo_status_logs
WHERE cargo_id = 'SH-HZ-20260629-0291'
ORDER BY created_at DESC;
```

| 顺序 | 内容 | 状态 |
|---:|---|---|
| 1 | `users` 登录、JWT、`/users/me` | 已打通 |
| 2 | `vehicles` 查询列表、详情 | 当前正在做 |
| 3 | `vehicles` 新增、修改、删除 | 下一步 |
| 4 | `cargo` 创建、查询、详情 | 后续 |
| 5 | `cargo_vehicle_binding` 绑定、解绑 | 后续 |
| 6 | `cargo_status_logs` 状态流转日志 | 后续 |
