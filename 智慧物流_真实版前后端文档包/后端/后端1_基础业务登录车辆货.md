# 后端 1 · 基础业务、登录、车辆、货物、绑定

## 1. 你的定位

你是业务底座负责人。后端 2 的 GPS、后端 3 的告警调度、两个前端页面都依赖你的基础数据。

---

## 2. 负责范围

### 必须负责

- Spring Boot 项目基础结构配合搭建。
- PostgreSQL 业务表：用户、车辆、货物、绑定、状态日志。
- 登录、刷新 token、登出、当前用户。
- 固定 5 类角色：`SHIPPER / DISPATCHER / WAREHOUSE / ADMIN / DRIVER`。
- 车辆新增、查询、详情、修改、删除。
- 货物创建、查询、详情。
- 货物车辆绑定、解绑。
- 货物状态上报。
- Seed 演示数据。
- 给后端 2/3 提供业务查询 service。

### 不负责

- 不负责 MQTT 订阅。
- 不负责 Kafka 消费。
- 不负责 GPS 入 TimescaleDB。
- 不负责 WebSocket 推送。
- 不负责告警算法。
- 不负责 RAG。

---

## 3. 你要实现的接口

后端启动后可以直接用 Swagger/Knife4j 调试：

```txt
http://localhost:8080/doc.html
```

操作顺序：

```txt
1. 页面应只显示「智慧物流接口」。
2. 调 POST /api/v1/auth/login，使用 dispatcher / 123456。
3. 复制返回的 data.accessToken。
4. 调 /users/me、/vehicles、/cargo 等接口时，在 Authorization 请求头填写 Bearer <accessToken>。
```

## 4. Service 方法必须给别人用

```java
Vehicle getVehicleByPlate(String plate);
Vehicle getVehicleByVinTopic(String vinTopic);
Vehicle getVehicleByImei(String imei);
Cargo getCargoById(String cargoId);
Cargo getActiveCargoByPlate(String plate);
Binding getActiveBindingByPlate(String plate);
Binding getActiveBindingByCargoId(String cargoId);
boolean isVehicleAvailable(String plate);
```

后端 2 要用这些方法把 MQTT Topic 和 GPS 点关联到车辆和货物。  
后端 3 要用这些方法生成告警、下发指令、查司机信息。

## 6. 6 天任务

### Day 1

- 建表。
- Seed 用户、车辆、货物。
- `/auth/login`。
- `/users/me`。

验收：5 个演示账号能登录。

### Day 2

- `/vehicles`、`/vehicles/{id}`、`POST /vehicles`。
- `/cargo`、`/cargo/{cargoId}`、`POST /cargo/bind`。
- 给后端 2 提供 `vinTopic -> plate -> cargoId` 查询。

验收：前端管理页能新增车辆、绑定货物。

### Day 3

- `PUT /vehicles/{id}`。
- `GET /vehicles/{plate}/active-task`。
- 修前端字段。

验收：后端 2 能通过车辆查当前货物。

### Day 4

- `POST /cargo/unbind`。
- `PUT /cargo/{cargoId}/status`。
- `DELETE /vehicles/{id}`。
- 基础权限拦截。

验收：货物状态能从 CREATED/LOADED/IN_TRANSIT/DELIVERED 流转。

### Day 5

- 配合全链路联调。
- 补数据一致性校验。
- 给演示准备稳定数据。

### Day 6

- 不加新功能。
- 修 bug。
- 写基础业务 README。

---

## 7. 注意事项

1. 车牌展示和 MQTT Topic 车牌必须分开保存。
2. 所有返回字段按接口文档，不要自己简写。
3. 不要把 seed 数据写死在 Controller。
4. 绑定关系一定要保证同一时间 1 车 1 货。
5. 货物状态流转要有日志，方便时间线展示。
